Manajemen User Di Mikrotik

Written by [Nur Hamim] with â™¥ on July 6, 2019 in NETWORKING

Apa itu User Management ?

User Management untuk melindungi Router Mikrotik kita, supaya tidak semua orang bisa masuk atau login serta mengkonfigurasi router. Karena dengan adanya User Management ini kita dipermudah untuk mengelola akses masuk ke system Router, untuk masuk ke system kita harus menggunakan user dan password terlebih dahulu.

Pada dasarnya RouterBoard Mikrotik memiliki User Management (User login) Default yaitu : User (Login)=admin dan Password=(kosong)

Kitu juga bisa membuat lebih dari 1 user bahkan banyak userpun bisa sesuai dengan kebutuhan, Misalkan kita punya rekan untuk meng-handle jaringan kita, kita bisa membuatkan user management dengan hak akses Full/Write/Read.

Berikut ini perbedaan hak ases Full/Write/Read Pada User Management :

  • Full

Hak akses ini merupakan hak akses full yang berada di paling atas atau tingkatan paling atas. User yang mempunyai hak akses full ini dia bisa menambahkan user, membaca, dan menulis atau write, serta menghapus konfigurasi, export, backup sistem mikrotik.

  • Write

Hak akses ini tidak jauh berbeda dengan hak akses full, yang berbeda dengan hak akses full di Write dia tidak bisa menambahkan user baru, dan juga tidak bisa melakukan konfigurasi backup konfigurasi mikrotik.

  • Read

Hak akses ini hanya bisa melihat dan memonitoring dan tidak bisa menabah konfigurasi tidak bisa mengedit konfigurasi mikrotik.

Berikut ini cara untuk menambahkan user baru dengan Akse Write dan Read :

  1. Klik system -> User -> add (+)
  2. Kita isikan name=userhamim (bebas) Group=Write Password=(bebas)

Sekarang kita membuat user dengan akses Read

  1. Klik system -> User -> add (+)
  2. Kita isikan name=santren (bebas) Group=Read Password=(bebas)

Kita juga bisa meng-costume akses user dengan cara membuat Group, dan nantinya kita bisa meng-costum sesuai dengan keinginan kita .

Berikut ini cara untuk membuat Group Untuk User :

  1. Klik system -> User -> Group -> Add (+)
  2. Isi name=Santren-Koding (bebas), Policies pilih atau Ceklis sesuai kebutuhan dengan kebutuhan user .

Simpan dengan klik Apply dan OK.

Berikutnya kita akan membuat user dengan Group akses Santren-Koding, cara nya sama seperti membuat user sebelumnya, bedanya Groupnya akan kita isi dengan Group yang barusan kita buat (Santren-Koding).
